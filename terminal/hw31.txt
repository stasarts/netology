8) man bash:
	- длину журнала history можно задать переменной: HISTFILESIZE (строка 846 man bash).
	- ignoreboth - значение для переменной HISTCONTROL (лист директив управления сохранением команд в истории). Cокращение от директив ignorespace (строки, начинающиеся с пробелов не сохраняются в истории) и ignoredups (строки, которые совпаают с предыдущей записью в истории не сохраняются).

9) Cкобки {} являются зарезервированными словами и должны встречаться там, где разрешено распознавать зарезервированное слово (строка 260 man bash).
	- строка 1035 man bash: в контексте Arrays. Фигурные скобки необходимы, чтобы избежать конфликтов с расширением пути.
	- строка 1091 man bash: в контексте Brace Expansion (это механизм, с помощью которого могут создаваться произвольные строки). Правильно сформированное Brace Expansion должно содержать открывающие и закрывающие фигурные скобки без кавычек.
	- строка 1197 man bash: в контексте Parameter Expansion. Фигурные скобки требуются, когда параметр является позиционным параметром с более чем одной цифрой или когда за параметром следует символ, который не следует интерпретировать как часть его имени. Параметр является параметром оболочки или ссылкой на массив (массивы).
	
10) Создать однократным вызовом touch 100000 файлов:
$ touch file{1..100000}
Создать 300000 файлов не удастся, т.к. есть ограничение по количеству принимаемых аргументов:
$ touch file{1..300000}
-bash: /usr/bin/touch: Argument list too long

11) Конструкция [[ -d /tmp ]]: возвращает True, если /tmp существует и является директорией. (строка 1843 man bash).

12) Команды, которые позволили вам добиться указанного вывода:
# Создадим вложенную директорию
$ mkdir -p /tmp/new_path_directory
# Создадим симлинк на баш в новой директории
$ ln -s /usr/bin/bash /tmp/new_path_directory
# Добавим новый каталог в переменную PATH в начало
$ PATH=/tmp/new_path_directory:${PATH}
# Проверка результата
$ type -a bash
bash is /tmp/new_path_directory/bash
bash is /usr/bin/bash
bash is /bin/bash

13) Отличие планирование команд с помощью batch и at:
at - выполняет команды в указанное время.
batch - выполняет команды, когда позволяют уровни загрузки системы; другими словами, когда среднее значение нагрузки падает ниже 1,5 или значения, указанного в вызове atd.

14) Завершение работы виртуальной машины:
vagrant suspend - с сохранением состояния (статус в virtualbox "saved")
vagrant halt - штатным образом (статус в virtualbox "Powerd Off")
